# SemanticStub エージェントガイド

SemanticStub は **.NET 10** で構築された OpenAPI ベースのスタブサーバーです。

## コア契約と挙動
- ベースとなる API 契約には **OpenAPI 3.1** を使用すること。
- API 定義は可能な限り OpenAPI 準拠を維持すること。
- カスタム挙動は **`x-stub-*` 拡張**で実装すること。
- リクエストマッチングは次の 2 モードをサポートすること。
  - **Strict モード**: HTTP メソッド、パス、Content-Type、リクエストスキーマを検証する。
  - **Hybrid モード**: 構造的な厳密性は保ちつつ、指定したリクエストボディ項目の柔軟な一致を許可する。

## アーキテクチャ原則
- モジュール化され、拡張しやすい構成を維持すること。
- シンプルで読みやすい C# を優先すること。
- 初期バージョンでは過度な設計を避けること。

## プロジェクト構成
- `SemanticStub.Core`: 共通モデルと抽象化。
- `SemanticStub.OpenApi`: OpenAPI の解析とマッピング。
- `SemanticStub.Matching`: マッチングロジック。
- `SemanticStub.Api`: ASP.NET Core によるホスティング。

## 開発ガイドライン
- 責務はプロジェクト境界ごとに分離すること。
- パッケージ管理は `Directory.Packages.props` に集約すること。
- 解析・マッチング・API 挙動に対するテストを書くこと。
- 段階的な実装を優先すること。
